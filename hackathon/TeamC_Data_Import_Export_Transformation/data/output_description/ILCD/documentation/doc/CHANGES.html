Release 1.1 (schemas build 944, tools build 985)

Changes since ELCD Release 1.0.1

Common Schemas/All Datasets

The ILCD format is a further development starting from the former ELCD format, which is discontinued. All identifiers point to "ILCD" now.

Extensions mechanism

  • While the formatExtensions construct is obsolete, each complex type may now contain an optional common:other element which in turn may contain content from arbitrary namespaces other than the ILCD ones. In addition, attributes from arbitrary other namespaces are allowed on each complex type.

Schema refactorings

  • Quite a few constructs (mainly from the Common_Groups which were previously chameleon) have been moved to the common namespace (see below).

Constraints

  • Some unique and keyref constraints that were not enforced have been added or corrected.

CASNumber SimpleType

  • The CAS number datatype now has a more restrictive pattern, the contents has to match the 000000-00-0 pattern (leading zeros are required).

Classification

  • categoryInformation is now classificationInformation, new structure as follows: 

       <classificationInformation>
      <common:classification [name="<name of classification system>"] [classes="<URI of classes document>"]>
         <common:class level="0">...</common:class>
         <common:class level="1">...</common:class>
      </common:classification>
   </classificationInformation>
  • The name and classes attributes can be omitted, in that case the default ILCD classification is used. The specified URI will be used to validate the classification information against that document.

@xml:lang

  • The xml:lang attribute can now be omitted (in this case it defaults to "en").

Review section

  • new element dataQualityIndicators on review section, see documentation for details
  • all reviewDetailsOn* elements are now obsolete and replaced by one single common:reviewDetails element
